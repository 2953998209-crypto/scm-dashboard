# 供应链智能看板系统

基于浏览器的供应链数据可视化智能分析看板，支持Excel数据上传、智能状态分析、图表可视化。

## 功能特性

- **Excel数据上传**：智能识别供应链、需求、销量、库存、配送周期等字段
- **智能状态分析**：自动计算满足率、库存天数，判定健康/正常/预警/紧急状态
- **图表可视化**：销量分布柱状图、需求销量对比图、状态占比饼图、配送周期趋势图
- **在线编辑**：点击表格单元格直接修改数据，自动重新分析
- **数据持久化**：本地存储，刷新不丢失
- **导出功能**：支持导出Excel数据

## 快速开始

### 方法一：本地运行

直接用浏览器打开 `index.html` 即可运行，无需安装任何依赖。

### 方法二：本地服务器运行

```bash
# 使用Python（推荐）
python -m http.server 8000

# 使用Node.js
npx serve
```

访问 http://localhost:8000

## 部署到GitHub Pages

### 方式一：自动部署（推荐）

1. 创建GitHub仓库，将代码推送到 `main` 分支
2. 仓库设置 -> Pages -> 来源选择 `GitHub Actions`
3. 推送代码后，GitHub Actions会自动部署

### 方式二：手动部署

1. 创建GitHub仓库
2. 将代码推送到仓库
3. 仓库设置 -> Pages -> 来源选择 `main` 分支，文件夹选择 `/ (root)`
4. 等待几分钟，访问 `https://<username>.github.io/<repo-name>/`

## Excel数据格式要求

系统支持智能识别以下字段（任意顺序）：

| 字段类型 | 支持的表头名称 |
|---------|--------------|
| 供应链 | 供应链、供应商、渠道、客户、经销商 |
| 产品 | 产品、商品、货品、食盐品种、品类 |
| 需求量 | 需求、需求量、需求数量、订单需求 |
| 销量 | 销量、销售量、销售额、实际销量 |
| 库存 | 库存、库存量、现有库存 |
| 配送周期 | 配送周期、周期、交货周期、配送天数 |
| 区域 | 区域、地区、片区、省份、城市 |

## 智能状态判定规则

| 状态 | 判定条件 |
|-----|---------|
| 健康 | 满足率≥95% 且 库存天数≥配送周期×1.5 |
| 正常 | 满足率≥85% 且 库存天数≥配送周期 |
| 预警 | 满足率<85% 或 库存天数<配送周期 |
| 紧急 | 满足率<70% 或 库存天数<配送周期×0.5 |
